# Color Palettes Reference

Pre-built color systems for quick project starts. Each palette includes full CSS custom properties ready to copy.

## How to Use

1. Pick a palette that matches your project's mood
2. Copy the CSS variables into your `:root`
3. Adjust individual values if needed — the palette gives you a cohesive starting point

## Gradient Backgrounds

For hero sections and feature areas. Apply to large sections, not the whole page.

```css
/* Subtle mesh gradient (dark) */
.hero-gradient-dark {
  background: 
    radial-gradient(ellipse at 20% 50%, rgba(99, 102, 241, 0.15) 0%, transparent 50%),
    radial-gradient(ellipse at 80% 20%, rgba(6, 182, 212, 0.1) 0%, transparent 50%),
    radial-gradient(ellipse at 60% 80%, rgba(168, 85, 247, 0.08) 0%, transparent 50%),
    var(--color-bg);
}

/* Warm gradient (light) */
.hero-gradient-warm {
  background:
    radial-gradient(ellipse at 0% 0%, rgba(251, 191, 36, 0.08) 0%, transparent 50%),
    radial-gradient(ellipse at 100% 100%, rgba(244, 63, 94, 0.06) 0%, transparent 50%),
    var(--color-bg);
}
```

## Usage Rules

1. **Pick ONE palette** per project. Don't mix palettes.
2. **Adjust 1-2 values** to make it your own. The palette is a starting point, not a prison.
3. **Test in context.** Colors look different on different backgrounds. Always check contrast.
4. **Primary color usage:** Use sparingly — for CTAs, links, active states. If everything is primary-colored, nothing is.
5. **Muted text:** Use for secondary information, timestamps, labels. Never for important content.
